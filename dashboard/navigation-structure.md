# Dashboard Navigation Structure

## Navigation Rail — Labels and Timing

```
Section ID    Label                    Time Guide    Sub-sections
──────────    ─────────────────────    ──────────    ────────────
01            Welcome + Agenda         3 min         —
02            What We Know             10 min        audit / focus groups
03            The Bridge               5 min         —
04            The Framework            20 min        4a: Infrastructure
                                                     4b: Protocol + PD
05            Discussion               10 min        —
06            Decisions                7 min         —
07            Next Steps               5 min         —
```

## Section 04 Sub-Navigation

Section 04 is the longest and most content-dense. It requires internal navigation between the two pillars. Both should be accessible directly from the nav rail (as indented sub-items) and from within Section 04 itself via a tab or toggle.

```
04  The Framework
    └── 4a  Pillar 1 — Infrastructure
    └── 4b  Pillar 2 — Protocol + PD
```

## State Indicators

Three visual states for nav items:
- **Upcoming:** muted label, no indicator
- **Current:** highlighted (accent color), bold label
- **Visited:** subdued check or dot indicator

These are informational only — facilitator can navigate to any section at any time regardless of state.

## Quick-Access Elements

The following should be accessible from anywhere in the dashboard without navigating to a specific section:

| Element | Trigger | Display |
|---|---|---|
| Sample unit plan template | Button in 4b / accessible globally | Full-screen overlay or side panel |
| Audit artifact | Reference note in Section 02 | Reminder to use printed prop |
| Time guide | Always visible in nav | Suggested minutes per section |

## Flow Logic Notes

- No forced linearity — any section is clickable at any time
- Section 04 defaults to 4a (Infrastructure) on first visit; 4b accessible via sub-nav
- Section 05 (Discussion) content is minimal by design — the facilitator is in conversation mode
- Section 06 (Decisions) contains visual selectors but no form submission — purely for visual anchoring
